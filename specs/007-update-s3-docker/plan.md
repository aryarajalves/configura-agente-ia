## Summary

This feature standardizes storage environment variables to generic S3 naming conventions, replaces the Redis-based infrastructure (TaskIQ broker and Pub/Sub bus) with RabbitMQ, and completely removes Celery and Flower from the project. It also configures the local Docker environment to connect to an external PostgreSQL instance via the `network_swarm_public` network.

## Technical Context

**Language/Version**: Python 3.11
**Primary Dependencies**: FastAPI, TaskIQ, taskiq-aio-pika, aio-pika, SQLAlchemy, boto3
**Storage**: PostgreSQL + pgvector, S3-compatible Storage (B2)
**Testing**: pytest
**Target Platform**: Linux (Docker)
**Project Type**: web-service (Backend/Infra)
**Performance Goals**: N/A (Standardization and infrastructure consolidation)
**Constraints**: Zero usage of Redis, Celery, or Flower. Mandatory RabbitMQ usage for tasks and events.
**Scale/Scope**: Repository-wide environment and infrastructure refactoring.
